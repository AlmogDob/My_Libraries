\doxysubsection{Detailed Description}
A single-\/header C library for simple 2D matrix operations on doubles and uint32\+\_\+t, including allocation, basic arithmetic, linear algebra, and helpers (LUP, inverse, determinant, DCM, etc.). 


\begin{DoxyItemize}
\item Storage is contiguous row-\/major (C-\/style). The element at row i, column j (0-\/based) is located at {\ttfamily elements\mbox{[}i $\ast$ stride\+\_\+r + j\mbox{]}}.
\item Dense matrices of {\ttfamily double} are represented by \mbox{\hyperlink{structMat2D}{Mat2D}}, and dense matrices of {\ttfamily uint32\+\_\+t} are represented by \mbox{\hyperlink{structMat2D__uint32}{Mat2\+D\+\_\+uint32}}.
\item Some routines assert shape compatibility using MATRIX2\+D\+\_\+\+ASSERT.
\item Random number generation uses the C library {\ttfamily rand()}; it is not cryptographically secure.
\item Inversion is done via Gauss-\/\+Jordan elimination with partial pivoting only when a pivot is zero; this can be numerically unstable for ill-\/conditioned matrices. See notes below.
\item To compile the implementation, define MATRIX2\+D\+\_\+\+IMPLEMENTATION in exactly one translation unit before including this header.
\end{DoxyItemize}

Example\+: \#define MATRIX2\+D\+\_\+\+IMPLEMENTATION \#include \char`\"{}matrix2d.\+h\char`\"{}

\begin{DoxyNote}{Note}
This one-\/file library is heavily inspired by Tsoding\textquotesingle{}s nn.\+h implementation of matrix creation and operations\+: \href{https://github.com/tsoding/nn.h}{\texttt{ https\+://github.\+com/tsoding/nn.\+h}} and the video\+: \href{https://youtu.be/L1TbWe8bVOc?list=PLpM-Dvs8t0VZPZKggcql-MmjaBdZKeDMw}{\texttt{ https\+://youtu.\+be/\+L1\+Tb\+We8b\+VOc?list=\+PLp\+M-\/\+Dvs8t0\+VZPZKggcql-\/\+Mmja\+Bd\+ZKe\+DMw}}
\end{DoxyNote}
\begin{DoxyWarning}{Warning}
Numerical stability\+:
\begin{DoxyItemize}
\item There is a set of functions for minors that can be used to compute the determinant, but that approach is factorial in complexity and too slow for larger matrices. This library uses Gaussian elimination instead.
\item The inversion function can fail or be unstable if pivot values become very small. Consider preconditioning or using a more robust decomposition (e.\+g., full pivoting, SVD) for ill-\/conditioned problems. 
\end{DoxyItemize}
\end{DoxyWarning}
